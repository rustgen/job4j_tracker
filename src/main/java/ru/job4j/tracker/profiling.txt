Анализ был проведён с помощью jconsole

При запуске программы было уже порядка 100 тысяч заявок, heap memory был в районе 180 МВ

При создании большого кол-ва заявок heap memory увеличилась почти в 2 раза, стала в районе 330 МВ

При удалении всех заявок heap memory то увеличивался, то уменьшался на графике и по окончании удаления
всех заявок стал около 400 МВ

При перегрузке памяти программа экстренно завершила работу с ошибкой
